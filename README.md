# SYSTEM

自分専用のステータス・クエスト管理アプリ（PWA）。ステータスウィンドウ、レベル/EXP、
デイリークエスト、期限超過ペナルティ、通知機能を備えています。

## ファイル構成

```
system-app/
├── index.html   ← アプリ本体（UIとロジック）
├── manifest.json ← ホーム画面インストール用の設定・アイコン指定
├── sw.js         ← サービスワーカー（オフライン化・通知）
├── db.js         ← 状態保存用の共通コード（index.htmlとsw.jsで共有）
└── icons/        ← アプリアイコン一式
```

## なぜ「スマホだけ」に置く方法が必要か

ホーム画面に専用アイコンで追加し、アプリを閉じていても通知を受け取れるようにするには、
ブラウザの仕様上、実際に `https://` で配信されたページとして開く必要があります。
Claudeのチャット内だけで完結させることはできないため、以下のどれかで自分専用に配信してください。
どれも他人には公開されず、URLを知っている人（＝自分だけ）が使う想定で問題ありません。

## 手順（GitHub Pagesが一番簡単）

1. GitHubで新しいリポジトリを作成
2. このフォルダの中身（index.html, manifest.json, sw.js, db.js, icons/）をそのままリポジトリ直下にアップロード
3. リポジトリの Settings → Pages → Source を "Deploy from a branch" にして、`main` ブランチ / `/ (root)` を選択
4. 数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` でアクセス可能になります
5. そのURLをiPhone/Androidでブラウザ（Safari / Chrome）で開く

> **訂正**：GitHubの無料（Free）個人プランでは、Privateリポジトリで GitHub Pages を使うことはできません。
> 上記の手順を使うにはリポジトリをPublicにする必要があります（内容は個人用クエストツールで、
> 機密情報は含まれないため実害はほぼありません）。ソースコードもPrivateのまま保ちたい場合は、
> 次のいずれかを検討してください。
>
> - **GitHub Pro**（有料・月$4程度）にアップグレードすると、Privateリポジトリのまま Pages が使えます
> - **Netlify** や **Cloudflare Pages**（無料）は、Privateリポジトリと連携したまま自動デプロイできます。
>   サイトのURL自体は公開されますが、ソースコードは非公開のままにできます

## ホーム画面に追加する

**Android（Chrome）**
1. URLを開く → 右上のメニュー → 「ホーム画面に追加」または「アプリをインストール」
2. 専用アイコンでホーム画面に追加されます

**iPhone（Safari）**
1. URLをSafariで開く（Chromeでは不可、必ずSafari）
2. 共有ボタン → 「ホーム画面に追加」
3. 専用アイコンでホーム画面に追加されます

## 通知について（正直な制約）

アプリ内の「設定」タブで通知を許可すると、アプリを開いている間は期限接近・超過を通知します。
ただし、アプリを完全に閉じた状態でも毎日確実に届く通知（バックグラウンドプッシュ通知）は、
ブラウザ標準の技術だけでは以下の制約があります。

- **Android（Chrome）**: インストール済みPWAで利用頻度が高いと、定期的なバックグラウンドチェック
  （Periodic Background Sync）が動く場合がありますが、発火タイミングはブラウザ任せで保証されません。
- **iPhone（Safari）**: バックグラウンドでの定期チェックは非対応です。アプリを開いたときにだけ、
  未達成・超過分をまとめて反映します。

**確実に「毎日必ず」通知を受け取りたい場合**の現実的な追加手段として、
[ntfy.sh](https://ntfy.sh) のような無料のシンプルなプッシュ通知サービスと、
GitHub Actions の定期実行（cron）を組み合わせる方法があります。仕組み：

1. ntfy.shアプリをスマホにインストールし、自分だけの合言葉的なトピック名を購読
2. GitHubリポジトリに、毎日決まった時刻に `https://ntfy.sh/<トピック名>` へ
   HTTPリクエストを送るだけのGitHub Actionsワークフローを設置（サーバー不要・無料）
3. アプリを開いていなくても、決まった時刻に確実に通知が届く

必要であれば、この ntfy.sh 連携部分（ワークフローファイル一式）も作成できます。

## データについて

- 保存先は端末内のIndexedDB（サービスワーカーからも読み書きできるようにするため、
  localStorageではなくこちらを採用しています）
- 「設定」タブからJSONでエクスポート／インポート可能
- 機種変更時はエクスポートしたJSONを新しい端末でインポートしてください

## カスタマイズ

- クエストの報酬EXP・報酬ステータス・ペナルティEXP・ペナルティステータス・報酬アイテムは
  クエストごとに自由に設定可能
- ペナルティの厳しさ全体は「設定」タブの倍率（0.5〜2倍）で調整可能
- ステータス名（STR/VIT/INT/FOC/PER）も「設定」タブから自分の好きな呼び方に変更可能

## AIによるクエスト自動発行（提案ではなく強制発行）

「クエスト」タブの「🤖 発行を要求」ボタン、または1日1〜2回のランダムな時刻に、SYSTEMが新しいクエストを
**そのままアクティブな状態で**発行します。原作同様、承認・却下のプロセスはありません
（気に入らなければ「削除」で消すことはできますが、選んで採用するものではありません）。

- 「行動ログ」欄に自分で今日やったこと・気分を記録しておくと、AIがそれを踏まえた内容を生成します
- AIが設定されていない、または呼び出しに失敗した場合は、内蔵の「緊急クエスト」プール（水分補給、運動、学習など）から自動で選ばれます。AI設定は必須ではありません
- 「設定」タブの AI QUEST ISSUANCE でプロバイダを設定できます
  - Anthropic (Claude) / OpenAI互換（Fujitsu ChatAI等のカスタムエンドポイントも可）/
    Google Gemini / Ollama（ローカル）に対応
  - APIキーは端末内のIndexedDBにのみ保存され、リクエストはブラウザから各プロバイダへ直接送信されます
  - Ollamaを使う場合は、スマホからOllamaサーバーに到達できるネットワーク（同一Wi-Fi等）が必要です
- ランダム発行はアプリの実行中（フォアグラウンド／直近までバックグラウンド）に有効なタイマーで動作します。
  Android/Chromeでは、インストール済みPWAとしての利用状況によってはバックグラウンド（Periodic
  Background Sync）でも稀に発行されることがありますが、iOSではこの経路は動作しません

## 自動難易度スケーリング

同じデイリークエストを3回連続で達成すると、SYSTEMが自動的に報酬EXPとペナルティEXPを引き上げます
（要求水準の上昇）。この挙動はクエスト編集画面の「自動スケーリング／ペナルティチェーンを有効にする」
チェックボックスでクエストごとにオフにできます。

## ペナルティチェーン

デイリークエストの期限を過ぎると、通常のEXP/ステータス減少に加えて、SYSTEMが**ペナルティクエスト**
（元のクエストより厳しい条件・数時間以内の短い期限）を強制発行します。これも達成できなければ、
さらに厳しい条件のペナルティクエストへとエスカレートします（最大3段階）。3段階目も失敗した場合は
SYSTEMが強制的にチェーンを鎮静化し、それ以上は悪化しません。

現実の体調不良時などのための安全弁として、ペナルティクエストには「休止を申請」ボタンがあります。
軽いペナルティは発生しますが、チェーンはそこで解除されます（無制限に使える無料の抜け道ではなく、
あくまで緊急時用です）。

## インベントリ

報酬アイテムを設定したクエストを達成すると、「インベントリ」タブにアイテムが追加されます。
レアリティ（コモン/レア/エピック/レジェンダリー）はクエスト編集画面で自由に設定できます。

## システムメッセージ

以下のタイミングで、システムからのメッセージボックスがポップアップします。
- クエストの期限を過ぎたまま日をまたいだとき（ペナルティ内容の一覧）
- アイテムを獲得したとき
- AIがクエストを提案したとき（採用するものを選択）
